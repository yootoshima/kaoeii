git clone กับ git pull ต่างกันยังไง

git clone

1. ใช้คัดลอก (ดาวน์โหลด) Repository จาก Remote มาไว้ในเครื่อง ครั้งแรก

2. สร้างโฟลเดอร์โปรเจกต์ใหม่ พร้อมตั้งค่า Remote (origin) ให้อัตโนมัติ

3. ใช้เมื่อยังไม่มีโปรเจกต์ในเครื่อง

git pull

1. ใช้ดึงการเปลี่ยนแปลงล่าสุดจาก Remote มาอัปเดต Repository ที่มีอยู่ในเครื่องแล้ว

2. ไม่สร้างโฟลเดอร์ใหม่ แต่จะอัปเดตโค้ดในโฟลเดอร์ปัจจุบัน

3. ใช้เมื่อเคย clone มาแล้ว

สรุป คือ

git clone = ดาวน์โหลดโปรเจกต์จาก Git มาไว้ในเครื่อง ครั้งแรก
git pull = ดึงโค้ดล่าสุดมาอัปเดตโปรเจกต์ที่มีอยู่ในเครื่องแล้ว


ยังไม่มีโปรเจกต์ในเครื่อง → ใช้ git clone
มีโปรเจกต์อยู่แล้ว แต่อยากได้โค้ดล่าสุด → ใช้ git pull
